# Security Policy — GTFS Express

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Report security issues by email to **security@weylandbinary.com**.

Please include:

- A description of the vulnerability and its potential impact
- Steps to reproduce (proof-of-concept, `curl` commands, screenshots)
- Affected version(s) or commit hash

We will acknowledge your report within **48 hours** and keep you informed of
progress. We kindly ask for a **7-day embargo** before public disclosure to give
us time to ship a fix.

Thank you for helping keep GTFS Express and its users safe.
